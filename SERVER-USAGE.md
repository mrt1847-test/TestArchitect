# 서버 사용 가이드

## 📋 서버는 선택사항입니다

**중요**: TestArchitect는 **서버 없이도 모든 기본 기능을 사용할 수 있습니다.**

### 서버 없이 사용 가능한 기능
- ✅ Recorder (테스트 녹화)
- ✅ Test Builder (Steps 생성)
- ✅ Code Generator (코드 생성)
- ✅ Runner (테스트 실행)
- ✅ 로컬 파일 저장

### 서버가 있을 때 추가되는 기능
- ✅ 테스트케이스 DB 저장
- ✅ 스크립트 DB 저장
- ✅ 실시간 동기화 (여러 클라이언트)
- ✅ 테스트 결과 저장
- ✅ 버전 관리

## 🚀 서버 사용 방법

### 서버 시작 (선택사항)

```bash
# 별도 터미널에서
npm run server
```

서버는 `http://localhost:3001`에서 실행됩니다.

### Electron 앱 실행

```bash
# 서버가 있어도 없어도 동일하게 실행
npm start
```

## 🔄 동작 방식

### 서버가 있는 경우
1. 앱 시작 시 서버에 자동 연결 시도
2. 연결 성공 시 DB 저장 및 실시간 동기화 활성화
3. Test Builder → 서버에 테스트케이스 저장
4. Code Generator → 서버에 스크립트 저장
5. Runner → 서버의 스크립트 목록 조회 가능

### 서버가 없는 경우 (로컬 모드)
1. 앱 시작 시 서버 연결 시도 (실패해도 계속 진행)
2. 로컬 파일 시스템으로 저장
3. Test Builder → 로컬 스토리지 + JSON 파일 저장
4. Code Generator → 로컬 파일로 저장
5. Runner → 로컬 `scripts/` 디렉토리에서 스크립트 조회

## 💡 권장 사용 시나리오

### 개인 사용 (서버 불필요)
- 서버 없이 사용
- 모든 기능 정상 작동
- 로컬 파일로 저장

### 팀 협업 (서버 권장)
- 서버 실행
- 여러 사용자가 동시에 작업
- 실시간 동기화
- 중앙 집중식 관리

## ⚠️ 주의사항

- 서버 연결 실패는 **에러가 아닙니다**
- 서버가 없어도 앱은 정상 작동합니다
- 콘솔에 경고 메시지가 나타나지만 기능에는 영향 없습니다

## 🔧 서버 상태 확인

앱 내에서 서버 상태를 확인하려면:

```javascript
// 개발자 도구 콘솔에서
await window.electronAPI.api.checkServer()
```

결과:
- `{ connected: true }`: 서버 연결됨
- `{ connected: false }`: 서버 연결 안 됨 (로컬 모드)

## 📝 결론

**서버는 선택사항입니다.**
- 기본 테스트: 서버 없이 사용 가능
- 팀 협업: 서버 실행 권장
- 모든 기능은 서버 없이도 작동합니다
